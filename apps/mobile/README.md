# BharatAgriLens — Mobile (Capacitor)

Capacitor wrapper that ships the existing `apps/web` React app as a
native Android (`.apk` / `.aab`) and iOS (`.ipa`) bundle. Single
codebase, two app stores.

## Architecture

```
apps/
├── web/                    React app — UI source of truth
│   └── dist/               built by `pnpm --filter @bal/web build`
│
└── mobile/                 Capacitor wrapper (this folder)
    ├── android/            Android Studio project (generated by Capacitor CLI)
    ├── ios/                Xcode project       (generated by Capacitor CLI)
    └── capacitor.config.ts
```

The web build's static assets land inside both native projects. Plugins
(`@capacitor/camera`, `@capacitor/preferences`) bridge JS to native
APIs.

## Bootstrap (one-time, after cloning the repo)

After `pnpm install` at the repo root:

```bash
cd apps/mobile

# Generate the Android Studio project
npx cap add android

# Generate the Xcode project  (MAC ONLY — skip on Windows/Linux)
npx cap add ios
```

This produces `apps/mobile/android/` and `apps/mobile/ios/` directories.
Commit both — they're the canonical native project files.

If you're on Windows or Linux and only have an Android target, you can
add the `ios/` directory later from a Mac. Capacitor handles each
platform independently.

## Daily dev loop

Make changes in `apps/web/src/**` as normal. Then sync to mobile:

```bash
# Builds the web app + syncs assets + plugins into both native projects
pnpm sync

# Or just one platform
pnpm sync:android
pnpm sync:ios
```

To open the native IDE (debug, inspect, run on device):

```bash
pnpm open:android    # opens Android Studio
pnpm open:ios        # opens Xcode  (Mac only)
```

To run on an attached device or running emulator/simulator:

```bash
pnpm run:android
pnpm run:ios         # Mac only
```

## What runs on what platform

| Code | Runs on | Notes |
|---|---|---|
| `apps/web/src/**` (React UI) | web + Android + iOS | identical |
| `apps/web/src/lib/platform.ts` | all three | platform detection |
| `apps/web/src/lib/camera.ts` | all three | branches: native Camera plugin or HTML `<input>` |
| `apps/api`, `apps/inference` | Railway only | mobile calls them via HTTPS |
| `services/training` | HF Spaces | irrelevant to mobile |
